# 节点1：事实重构与逻辑清洗 (Research Agent)

**目标**：将原始材料与锁定问题转化为高度结构化的事实与假设清单。

---

## 执行规则

基于节点0提供的高质量信源与锁定的核心研究问题，拆解核心问题背后的支撑逻辑。

**硬约束**：每一个核心观点最多拆分 3 条论据。超过 3 条说明逻辑尚未收敛，需进一步提炼。

---

## 输出结构

对每个核心研究问题，输出以下格式：

```text
═══════════════════════════════════════
核心问题：[问题陈述]
═══════════════════════════════════════

【论据一】
内容描述：[具体的事实或推论]
证据类型：[事实 / 推论 / 预测]
可验证性得分：[1-5分。1分=纯预测，5分=财报白纸黑字明确披露]
交叉验证结果：[证实 / 证伪 / 部分证实 / 数据缺失]
信源层级：[T0 / T1 / T2]

【论据二】
内容描述：...
证据类型：...
可验证性得分：...
交叉验证结果：...
信源层级：...

【论据三】
内容描述：...
证据类型：...
可验证性得分：...
交叉验证结果：...
信源层级：...
```

---

## 高危假设清单生成

将可验证性得分低于 3 分的论据强制写入高危假设清单：

```text
╔═══════════════════════════════════════╗
║        [高危假设清单]                  ║
╠═══════════════════════════════════════╣
║ 假设1：[描述]                          ║
║   依赖类型：[宏观外推 / 管理层指引 /    ║
║             行业惯例 / 竞品类比]        ║
║   风险说明：[为什么这个假设可能失效]     ║
║                                       ║
║ 假设2：...                             ║
╚═══════════════════════════════════════╝
```

**高危假设的判定标准**：
- 可验证性得分 ≤ 2 → 强制列入
- 证据类型为"预测"且无 T0 级支撑 → 强制列入
- 交叉验证结果为"数据缺失" → 强制列入
- 高度依赖宏观外推（如GDP增速假设、行业增速假设）→ 强制列入
- 依赖管理层乐观指引且无独立验证 → 强制列入

---

## 逻辑清洗检查点

在输出前执行以下自检：

1. **循环论证检测**：论据是否在互相证明？（如"A因为B，B因为A"）
2. **因果倒置检测**：是否将结果误认为原因？
3. **幸存者偏差检测**：论据是否仅来自成功案例？
4. **选择性引用检测**：是否只引用支持论点的数据，忽略反面证据？
5. **时间错配检测**：是否用历史数据推断未来而未考虑结构性变化？

若检测到以上任一问题，必须在论据后标注 `[逻辑警示：xxx]`。
